picGo+typora+github打造个人图床

Created2025-11-02|Updated2025-11-02
|Post Views:

你是否遇到图片在网页里加载不出来的情况？

对于 hexo 框架，就算是配置了 Typora 图片配置为保存到同名文件夹下，_confid.yml 也设置 post_asset_folder 为 true，甚至下了插件，也无济于事？

这里推荐以图床的方式解决这个问题

架构大致如下

首先， typora 的图片偏好我保持默认，即放到同目录下的 ./assets 的文件

接着是 picGo 下载与配置

https://github.com/Molunerfinn/PicGo

下载完成后，对图床设置的 github 内进行设置

这里 Github 的仓库需要自己提前建好

由于在使用适配 Typora 的 picGo 之前，了解了一下 picX，这里直接用 picX 一键配置

https://picx.xpoet.cn/#/config

https://github.com/settings/tokens/new 使用这个链接建立一个 token

丢到 picx 里，选择一键配置，仓库就建好了

仓库名默认叫做 picx-images-hosting，分支 master

然后就用此 token,用户名/仓库名,master 去 picGo 配置信息

最后添加一个自定义域名加速 https://cdn.jsdelivr.net/gh/<your_user_name>/picx-images-hosting

配置好后到上传区上传文件，成功后会自动复制其url

相册里也会新增内容

最后是 typora 与 picGo 适配

在文件偏好里设置

这样，当你复制图片到 typora 后，右击图片会有一个上传选项，点击就自动转化为了外链。

这样图床就配置好了，还是很方便。
